Manage Program Budgets

Any changes made after first awarding will be reviewed by VSAC before becoming effective.

Components

Detail block

Detail blocks are a custom nested grid using flexbox that’s designed to provide details for budgets.

An element with class .detail-block acts as a container for elements with class .details which in turn act as a container for elements with class .detail.

Design intent

Detail blocks are designed to behave responsively and be reasonably flexible, however, they were created with specific content in mind and therefore come with a few considerations and limitations.

A .detail-block can have any number of .details columns inside it, however, adding more will cause each individual column to shrink and get narrower. Typically a .detail-block can comfortably fit four to six .details columns in it depending on the content.

Similarly, a .details column can have any number of .detail blocks inside it, it’s been responsively designed to go from a column layout on large screen sizes to a row layout for small screen sizes. As a result, it’s best to only have up to two .detail blocks per .details column.

Formatting & accessibility

The table on the budget view uses the .table-collapsible class to better display the content for small screen users. Read more about collapsible table formatting.

For small screens the table’s <thead> is hidden, the <tbody> table cells are stacked vertically and table header labels are shown next to each cell using CSS. Each table cell will need a data-title attribute with the corresponding table heading in order to properly display the labels.

Table caption

Data tables require a <caption> element to describe the table content. This can be set to display only to screenreaders by adding the .sr-only class.

Table cell alignment

Text in table cells default to being left-aligned. Table cell alignment can be set on the <td> or <th> using Bootstrap’s text utility classes. Because this is using .table-collapsible, these alignments will be overridden for small screens with labels left aligned and the values right aligned.

Columns containing dollar amounts should be right aligned to make it easier to scan columns. Add the .text-right class to all <th> or <td> elements containing dollar amounts. Be sure to also set the .text-right class on the corresponding heading in the <thead> to align it with the values.
