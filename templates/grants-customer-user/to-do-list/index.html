Components

Page Title

Page title Subtitle

<!-- Page title -->
<h1 class="page-title mb-4">Page title <small>Subtitle</small></h1>

The page title has it’s own page-title class which controls the weight of the heading and offers some subtitle styling if needed using a nested <small> element.

Bottom margin is controlled using Bootstrap’s spacing utility classes, in this case mb-4 adds ~24px of bottom margin to the heading. This uses the utility classes in order to be more flexible depending on the content on the page.

Tasks block

The task list uses a variation of the <div class="block"> element used elsewhere. At its most basic level, it’s a <div> with a heading and two ordered lists: one for open tasks and one for completed tasks. The block--tasks class on the wrapping div provides the border styling and padding and the task-header class on the heading sets the size of the heading and the background color behind it.

Tasks

Each task list item <li class="task"> is made up of several components, some are required and some are optional depending on the task. Using Flexbox CSS each task is made up of three columns which will stack on top of each other for smaller screens.

Task numbering

The first column is a number or a checkmark (depending on whether the task has been marked completed or not). This is column is created entirely through CSS and is not visible in the HTML. Numbering is based on the position of the list item in the ordered list.

Task description

The second column is the task description <div class="task__description">. This always has a <div class="task__title"> containing the label for the task.

If there is help content or an extended description for a task, it can be added in a modal.

Additional descriptive content can be added in a paragraph after the title <div>.

Task link or status

The third and final column is the task link or status <div class="task__link">. This either contains a link styled as a button linking to where the user can complete their task or cotains text representing the status of the task. This can be left blank which will cause the task description to run the full width.

Additional screenreader only text can be added to either the button or status text by using a <span class="sr-only"> as shown below.

Completed task variant

A task can be marked completed by adding the task--completed class to the <li class="task">. This replaces the number styling with a checkmark and fades the text slightly until hovered.

Modals

This view can have any number of modal windows for the additional details relating to a task. The HTML for these modals should be included either after the closing </main> tag or the closing </footer> tag on the page. Each modal should have it’s own unique ID. See Bootstrap’s modal documentation for more details.
